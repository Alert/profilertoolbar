ProfilerToolbar v0.2.4

Profiler module for kohana framework

ProfilerToolbar it is another implementation of the DebugToolbar written by Aaron Forsander, but has additional features.

  Project on github: https://github.com/Alert/profilertoolbar

The module has three parts: HTML, Firebug and error page

Usage

  • Download project and put files into a folder called 'profiletoolbar' under modules
  • Enable Module. IMPORTANT! The ProfilerToolbar must be connected before modules of Cached and Database. Because the ProfilerToolbar replaces some files of them.
      Kohana::modules(array(
    ...
    'profilertoolbar' => MODPATH.'profilertoolbar',
    'cache'           => MODPATH.'cache',
    'database'        => MODPATH.'database',
    ...
  ));
  • Usage HTML output. In the main template write:
      <html>
  <body>
    ...
    content
    ...
    <?php ProfilerToolbar::render(true); ?>
  </body>
  </html>
  • Usage Firebug output. By default data display in each method of all controllers and you do not have to do anything. But if you want to disable this feature and show data only in some methods, set FALSE to config parameter "showEverywhere".
      ...
  // firebug data settings
  'firebug'=>array(
    'enabled'         => true, // if set FALSE, panel don't ...
    'showEverywhere'  => FALSE, // if set TRUE you don't need ...
  ...

    After that, in any method where you need to get the data, write:
      ProfilerToolbar::firebug();

    I recommend doing that in method after of each controller where you want see data:

      class Controller_Ajax extends Controller {
 
    public function after(){
      ...
      parent::after();
      ProfilerToolbar::firebug();
    }
 
    ...
    ...
    ...
  }
  • rejoice!

Example from screenshot
  $user = new stdClass();

  ProfilerToolbar::addData('test string');
  ProfilerToolbar::addData(rand(1, 1000)/ rand(1, 1000));
  ProfilerToolbar::addData($user);
  ProfilerToolbar::addData($this->request->headers());
  ProfilerToolbar::addData('other data','second tab');

Notice 3: You can disable HTML output in config file and use only Firebug data:

  return array(
    // rendered panel data settings
    'html'=>array(
      'enabled' => false, // if set FALSE, panel don't be rendered on page
      ...
    ),
    // firebug data settings
    'firebug'=>array(
      'enabled' => true, // if set FALSE, panel don't be shown in firebug
      ...
    ),
    ...
  }
